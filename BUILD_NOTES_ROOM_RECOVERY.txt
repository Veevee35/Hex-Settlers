Hex Settlers room-recovery build

Fixes:
- Every room-scoped client request now carries the active room code.
- The server can safely restore an authenticated player's socket-to-room binding before processing the request.
- This closes the reconnect race that left the lobby visible while actions returned "Not in a room."
- Recovery is limited to accounts already present in the requested room; it cannot be used to enter an unrelated room.

Validation:
- JavaScript syntax checks passed.
- TypeScript bootstrap compiled successfully.
- Full automated suite passed: 104/104 tests.
- Added an integration test that replaces the host WebSocket and performs a lobby action without an explicit rejoin; the room is recovered and the action succeeds.
